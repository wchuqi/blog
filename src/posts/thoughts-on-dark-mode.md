深色模式现在几乎是应用的标配，但做好它并不只是「把白底换成黑底」。

## 不要用纯黑

纯黑（`#000000`）配纯白文字（`#ffffff`）对比度过高，长时间阅读容易疲劳，在 OLED 屏上还会有拖影。更舒服的做法是用深灰，比如这个博客用的是 GitHub 的 `#0d1117`。

## 用 CSS 变量管理主题

核心思路是把所有颜色抽成 CSS 变量，挂在 `:root` 上，深色模式通过一个 `data-theme` 属性覆盖：

```css
:root {
  --bg: #ffffff;
  --text: #1f2328;
}

[data-theme='dark'] {
  --bg: #0d1117;
  --text: #e6edf3;
}
```

然后所有组件都引用变量，而不是写死颜色：

```css
body {
  background: var(--bg);
  color: var(--text);
}
```

切换主题时，只需要改 `<html>` 上的一个属性，整站颜色就会平滑过渡。

## 跟随系统，但允许覆盖

理想的体验是：

1. 用户第一次访问，跟随系统的 `prefers-color-scheme`
2. 用户手动切换后，记住选择（存到 `localStorage`）
3. 如果用户没手动选过，系统主题变化时实时跟随

这个博客的 `useTheme` hook 就是这么做的。

## 别忘了图片和代码块

深色模式下，过亮的图片会很刺眼。可以考虑给图片加一点点降低亮度的滤镜，代码高亮主题也要换成深色版本。细节决定体验。
